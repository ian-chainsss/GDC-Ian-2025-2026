# Opstellen & gebruiken van een database schema in PostgreSQL
In deze documentatie zal ik uitleggen hoe ik een database schema heb opgesteld en gebruikt in PostgreSQL voor mijn project. Ik zal de stappen beschrijven die ik heb genomen om de database te ontwerpen, te implementeren en te gebruiken in mijn applicatie.
## Database Schema Ontwerp
1. Nadenken over het opdelen van de database in verschillende schema's: Welke tables kan ik samenvoegen tot één overzichtelijk schema? Welke tabellen moeten gescheiden blijven?  
De tables `posts` en `comments` kunnen mooi samengebundeld worden in een `content` schema.
De tables `users`, `password_resets` en `refresh_tokens` kunnen mooi samengebundeld worden in een `access` schema.
2. Aanmaken van de schema's in PostgreSQL: Gebruik SQL-commando's om de schema's te creëren.  
```sql
CREATE SCHEMA content;
CREATE SCHEMA access;
```
3. Aanmaken van tables binnen de schema's: Definieer de structuur van elke tabel, inclusief kolommen, datatypes en eventuele constraints.  
```sql
CREATE TABLE "access"."users" (
  "id" INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  "username" varchar(100) UNIQUE NOT NULL,
  "email" varchar(350) UNIQUE NOT NULL,
  "password_hash" text NOT NULL,
  "role" int NOT NULL DEFAULT 9,
  "created_at" timestamptz NOT NULL DEFAULT (now()),
  "updated_at" timestamptz NOT NULL DEFAULT (now()),
  "last_login_at" timestamptz
);
```
De `"access"."users"` geeft aan dat de tabel `users` zich bevindt in het schema `access`.
4. Je kan de table in het vervolg altijd aanspreken met de prefix van het schema, bijvoorbeeld: `SELECT * FROM access.users;`
5. Gebruik met ORMs zoals SQLAlchemy in Python: Wanneer je een ORM gebruikt, zorg er dan voor dat je de schema's correct definieert in je modelklassen. Bijvoorbeeld, in SQLAlchemy kun je het `schema` argument gebruiken om aan te geven in welk schema de tabel zich bevindt.  
```python
class User(Base):
    __tablename__ = "users"
    __table_args__ = {"schema": "access"}
    
    id = Column(Integer, primary_key=True)
    username = Column(String(100), unique=True, nullable=False)
    email = Column(String(350), unique=True, nullable=False)
    password_hash = Column(Text, nullable=False)
    role = Column(Integer, nullable=False, default=9)
    created_at = Column(DateTime(timezone=True), server_default=func.now(), nullable=False)
    updated_at = Column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now(), nullable=False)
    last_login_at = Column(DateTime(timezone=True))
```
Bij deze class `User` wordt aangegeven dat de tabel `users` zich bevindt in het schema `access` door het gebruik van `__table_args__ = {"schema": "access"}`. Hierdoor weet SQLAlchemy dat het de tabel `users` moet zoeken binnen het `access` schema wanneer er queries worden uitgevoerd.
